## MySQL 실행 방법
윈도우 환경에서 MySQL을 처음부터 실행하고 접속하는 과정을 단계별로 기술한다. 이 절차는 향후 동일한 작업 시 매뉴얼로 활용 가능하다.



### 1. 환경 설정 (최초 1회 필수)
명령 프롬프트(CMD)에서 `mysql` 명령어를 즉시 인식하도록 경로를 등록하는 과정이다.

1.  `Win + R`을 누르고 `sysdm.cpl` 입력 후 엔터를 누른다.
2.  **[고급]** 탭 → **[환경 변수]** 버튼을 클릭한다.
3.  '시스템 변수' 목록에서 **Path**를 찾아 선택하고 **[편집]**을 누른다.
4.  **[새로 만들기]**를 누르고 MySQL의 bin 폴더 경로를 입력한다.
    * 기본 경로: `C:\Program Files\MySQL\MySQL Server 8.0\bin`
5.  모든 창의 **[확인]**을 눌러 설정을 저장한다.





### 2. 서버 실행 및 접속 (사용 시마다 수행)

#### **1단계: 서버 엔진 가동**
데이터베이스 서버가 백그라운드에서 실행 중이어야 접속이 가능하다.
1.  **관리자 권한**으로 CMD를 실행한다. (시작 메뉴 → 'cmd' 검색 → 마우스 우클릭 → 관리자 권한으로 실행)
2.  다음 명령어를 입력한다.
    ```cmd
    net start mysql
    ```
    * *참고: 이미 실행 중이라는 메시지가 뜨면 다음 단계로 넘어간다.*

#### **2단계: 클라이언트 접속 (로그인)**
서버에 접속하여 명령어를 입력할 수 있는 상태로 진입한다.
1.  CMD 창(일반 권한도 무방)에서 다음 명령어를 입력한다.
    ```cmd
    mysql -u root -p
    ```
2.  설치 시 설정한 root 비밀번호를 입력하고 엔터를 누른다. 보안상 비밀번호는 화면에 표시되지 않는다.
3.  `mysql>` 프롬프트가 나타나면 접속에 성공한 것이다.





### 3. 주요 관리 명령어 요약

| 구분 | 명령어 | 설명 |
| :--- | :--- | :--- |
| **서버 시작** | `net start mysql` | 엔진 가동 (관리자 권한 필수) |
| **서버 종료** | `net stop mysql` | 엔진 중지 (관리자 권한 필수) |
| **접속** | `mysql -u root -p` | 관리자 계정으로 로그인 |
| **DB 목록 확인** | `SHOW DATABASES;` | 생성된 데이터베이스 확인 |
| **DB 선택** | `USE [DB명];` | 특정 데이터베이스 활성화 |
| **종료** | `EXIT;` 또는 `QUIT;` | MySQL 클라이언트 종료 |



### 4. 주의 사항 및 오류 대처
* **명령어 오류:** 모든 SQL 명령어는 세미콜론(`;`)으로 끝나야 실행된다.
* **입력 취소:** `->` 표시가 뜨며 명령어가 멈추지 않을 때는 `\c`를 입력하고 엔터를 눌러 취소한다.

## 테이블 수정 방법

MySQL에서 기존 테이블을 삭제하고 다시 만들거나, 이미 존재하는 테이블의 구조를 변경하는 방법은 다음과 같다.



### 1. 테이블 삭제 후 다시 생성 (Recreate)
기존 테이블을 아예 없애고 새로 정의하고자 할 때 사용한다. 이 작업을 수행하면 **기존에 저장된 모든 데이터가 영구적으로 삭제**되므로 주의가 필요하다.

1.  **테이블 삭제:**
    ```sql
    DROP TABLE topic;
    ```
    * 만약 테이블 존재 여부를 확인하며 안전하게 삭제하려면 `DROP TABLE IF EXISTS topic;`을 사용한다.
2.  **테이블 재생성:**
    이전 답변에서 안내한 수정된 `CREATE TABLE` 구문을 다시 입력하여 실행한다.





### 2. 기존 테이블 구조 수정 (Modify)
데이터를 유지한 상태에서 컬럼을 추가, 삭제하거나 자료형을 바꾸고 싶을 때는 `ALTER TABLE` 명령어를 사용한다.

#### **A. 새로운 컬럼 추가 (ADD)**
`profile` 컬럼 뒤에 `phone` 컬럼을 추가하는 예시다.
```sql
ALTER TABLE topic ADD phone VARCHAR(20) NULL;
```

#### **B. 기존 컬럼의 자료형 변경 (MODIFY)**
`title` 컬럼의 길이를 100에서 200으로 늘리는 예시다.
```sql
ALTER TABLE topic MODIFY title VARCHAR(200) NOT NULL;
```

#### **C. 컬럼 이름 및 자료형 동시 변경 (CHANGE)**
`author`라는 컬럼 이름을 `nickname`으로 바꾸면서 자료형도 변경하는 예시다.
```sql
ALTER TABLE topic CHANGE author nickname VARCHAR(20) NULL;
```

#### **D. 컬럼 삭제 (DROP)**
`profile` 컬럼을 삭제하는 예시다.
```sql
ALTER TABLE topic DROP COLUMN profile;
```





### 3. 구조적 지적 및 주의사항 

* **데이터 유실 리스크:** `DROP TABLE`은 복구가 불가능한 삭제 명령어다. 중요한 데이터가 있다면 반드시 백업 후 실행해야 한다.
* **제약 조건 충돌:** `ALTER TABLE`로 컬럼의 자료형을 변경할 때, 이미 저장된 데이터가 변경하려는 자료형과 맞지 않으면 오류가 발생한다. (예: 문자가 들어있는 컬럼을 숫자로 변경 시도)
* **인덱스 영향:** 기본키(`PRIMARY KEY`)나 `AUTO_INCREMENT` 설정을 변경할 때는 다른 컬럼과의 연관 관계를 먼저 확인해야 한다.



### 4. 현재 상태 확인 방법
수정이 정상적으로 반영되었는지 확인하려면 다음 명령어를 입력한다.
```sql
DESC topic;
```
* 해당 테이블의 현재 컬럼 구성, 자료형, 키 설정 등을 표 형태로 확인할 수 있다.

## CRUD 

SQL에서 CRUD는 데이터를 관리하는 네 가지 기본 기능인 생성(Create), 조회(Read), 수정(Update), 삭제(Delete)를 의미한다. 각 기능에 대응하는 표준 SQL 명령어와 핵심 문법은 다음과 같다.



### 1. Create (생성): INSERT
테이블에 새로운 데이터를 삽입하는 단계다.

* **기본 문법:**
    ```sql
    INSERT INTO 테이블명 (컬럼1, 컬럼2, ...) VALUES (값1, 값2, ...);
    ```
* **특징:** `AUTO_INCREMENT`가 설정된 컬럼(예: id)은 값을 직접 입력하지 않아도 시스템이 자동으로 할당한다.
* **예시:**
    ```sql
    INSERT INTO users (name, email) VALUES ('Kim', 'kim@example.com');
    ```




### 2. Read (조회): SELECT
테이블에 저장된 데이터를 불러오는 단계다.

* **기본 문법:**
    ```sql
    SELECT 컬럼1, 컬럼2 FROM 테이블명 WHERE 조건;
    ```
* **주요 옵션:**
    * `*`: 모든 컬럼 조회.
    * `WHERE`: 특정 조건에 맞는 데이터만 필터링.
    * `ORDER BY`: 데이터 정렬 (ASC: 오름차순, DESC: 내림차순).
* **예시:**
    ```sql
    SELECT name FROM users WHERE id = 1;
    ```



### 3. Update (수정): UPDATE
기존에 저장된 데이터를 변경하는 단계다.

* **기본 문법:**
    ```sql
    UPDATE 테이블명 SET 컬럼1 = 값1, 컬럼2 = 값2 WHERE 조건;
    ```
* **주의사항:** **`WHERE` 절을 생략하면 테이블 내의 모든 데이터가 변경된다.** 반드시 수정할 대상을 특정해야 한다.
* **예시:**
    ```sql
    UPDATE users SET email = 'new_kim@example.com' WHERE id = 1;
    ```


### 4. Delete (삭제): DELETE
테이블의 데이터를 삭제하는 단계다.

* **기본 문법:**
    ```sql
    DELETE FROM 테이블명 WHERE 조건;
    ```
* **주의사항:** **`WHERE` 절을 생략하면 테이블의 모든 데이터가 삭제된다.** 구조는 남지만 데이터는 모두 소실되므로 매우 주의해야 한다.
* **예시:**
    ```sql
    DELETE FROM users WHERE id = 1;
    ```




### 5. CRUD 명령어 요약 및 비교

| 기능 | SQL 명령어 | 작업 대상 | 비고 |
| :--- | :--- | :--- | :--- |
| **Create** | `INSERT` | 행(Row) 추가 | 새로운 데이터 생성 |
| **Read** | `SELECT` | 행(Row) 조회 | 데이터 읽기 및 필터링 |
| **Update** | `UPDATE` | 행(Row) 수정 | 기존 데이터의 값 변경 |
| **Delete** | `DELETE` | 행(Row) 삭제 | 기존 데이터 제거 |



### 6. 참고 사항
* `UPDATE`와 `DELETE`를 실행하기 전에는 반드시 `SELECT` 문으로 삭제/수정 대상을 먼저 확인하는 습관을 들이는 것이 데이터 사고 방지를 위해 권장된다.




## MySQL 종료 방법

MySQL을 종료하는 방법은 현재 상태가 **클라이언트 접속 상태**인지, 아니면 **서버 엔진 자체를 중지**하려는 것인지에 따라 다르다.
---

### 1. MySQL 클라이언트 종료 (로그아웃)
`mysql>` 프롬프트가 뜨는 접속 상태에서 빠져나와 일반 명령 프롬프트(CMD)로 돌아가는 방법이다.

* **명령어 입력:** 다음 중 하나를 입력하고 엔터를 누른다.
    * `EXIT;`
    * `QUIT;`
    * `\q` (세미콜론 없이 바로 실행 가능)
* **결과:** `Bye`라는 메시지와 함께 접속이 끊기며 일반 CMD 상태로 복귀한다.



### 2. MySQL 서버 서비스 중지 (엔진 가동 중단)
윈도우 백그라운드에서 실행 중인 MySQL 엔진 자체를 완전히 끄는 방법이다. 더 이상 데이터베이스를 사용하지 않을 때 시스템 자원을 아끼기 위해 수행한다.

* **방법:** **관리자 권한**으로 실행된 CMD 창에서 다음 명령어를 입력한다.
    ```cmd
    net stop mysql
    ```
* **결과:** "MySQL 서비스를 잘 멈췄습니다"라는 메시지가 출력되면 서버가 종료된 것이다.



### 3. 종료 관련 명령어 요약

| 대상 | 명령어 | 실행 위치 |
| :--- | :--- | :--- |
| **클라이언트** | `EXIT;` 또는 `QUIT;` | `mysql>` 프롬프트 |
| **서버(서비스)** | `net stop mysql` | 윈도우 CMD (관리자) |
| **창 강제 종료** | `Ctrl + C` | 어느 곳에서나 가능 |



### 4. 주의사항
* **클라이언트 종료 ≠ 서버 종료:** `EXIT` 명령어로 클라이언트를 종료해도 MySQL 서버 서비스는 백그라운드에서 계속 실행 중이다. 서버까지 완전히 끄려면 반드시 `net stop` 명령을 별도로 수행해야 한다.
* **데이터 무결성:** 데이터를 입력(`INSERT`)하거나 수정(`UPDATE`) 중인 상태에서 강제로 창을 닫거나 서버를 중단하면 데이터가 유실되거나 테이블이 깨질 위험이 미세하게 존재한다. 가급적 정상적인 종료 절차를 밟는 것을 권장한다.
