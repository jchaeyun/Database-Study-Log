# section 1~3


## 1. 데이터베이스 기초 (섹션 1)
데이터베이스의 정의와 파일 시스템 대비 장점을 다룬다.

* **본질:** 데이터베이스는 단순히 저장하는 도구가 아니라, 원하는 데이터를 효율적으로 추출(인출)하기 위한 시스템이다.
* **파일과의 차이:** 파일 시스템은 직관적이나 데이터 중복과 관리 효율성 면에서 한계가 있다. 데이터베이스는 구조화를 통해 데이터 무결성을 유지하고 다수 사용자의 동시 접근을 제어한다.

## 2. MySQL 도입 및 설치 (섹션 2)
관계형 데이터베이스 관리 시스템(RDBMS)인 MySQL의 설치와 목적을 설명한다.

* **목적:** 대규모 데이터를 체계적으로 관리하고, 웹 서비스 등과 연동하여 동적인 데이터를 처리하기 위함이다.
* **환경 구축:** Windows, macOS 등 운영체제별 설치 과정을 수행하여 실습 환경을 조성한다.

## 3. MySQL 기본 구조와 접속 (섹션 3)
데이터가 저장되는 논리적 계층 구조와 접근 방법을 정의한다.

* **계층 구조:** **서버(Server) > 스키마(Schema) > 테이블(Table)** 순으로 구성된다.
    * **서버:** 시스템 전체를 구동하는 엔진이다.
    * **스키마:** 연관된 테이블들을 그룹화한 데이터베이스 폴더 개념이다.
    * **테이블:** 행(Row)과 열(Column)로 구성된 실제 데이터 저장 단위이다.
* **제어 수단:** SQL(Structured Query Language)을 통해 데이터베이스와 통신하며, 서버 접속 후 특정 스키마를 선택하여 작업을 수행한다.
